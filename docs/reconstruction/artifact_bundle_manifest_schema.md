# Artifact Bundle Manifest Schema

## Purpose

`artifact_bundle_manifest.json` describes repo-relative frozen evidence files that are required to validate or re-audit the reconstruction without storing large artifacts in normal Git.

The minimum template is stored at:

`artifacts/final_graph/selected_final_graph/rebuild/artifact_bundle_manifest.minimum.template.json`

## Top-Level Fields

| Field | Type | Required | Description |
|---|---|---:|---|
| `schema_version` | string | yes | Manifest schema identifier. Current value: `kg-reconstruction-artifact-bundle-manifest-v1`. |
| `repo_commit` | string | yes | Git commit the manifest is intended to pair with. |
| `bundle_id` | string | yes | Stable human-readable bundle identifier. |
| `created_at_utc` | string | yes | UTC creation timestamp or template marker. |
| `artifacts` | array | yes | Artifact rows. |

## Artifact Row Fields

| Field | Type | Required | Description |
|---|---|---:|---|
| `path` | string | yes | Repository-relative artifact path. Paths with spaces must be preserved exactly. |
| `sha256` | string | yes | Expected SHA256 hash of the artifact file. |
| `size_bytes` | integer | yes | Expected file size in bytes. |
| `storage_class` | string | yes | One of `git`, `external`, `git_lfs`, `runtime_generated`, or `external_optional`. |
| `required_for_validate_only` | boolean | yes | Whether `run_frozen_artifact_reconstruction_audit.sh --validate-only` requires this file. |
| `required_for_force_rebuild` | boolean | yes | Whether wrapper regeneration with `--force` requires this file. |
| `required_for_thesis_provenance` | boolean | yes | Whether the file supports thesis provenance claims beyond the minimum validate-only contract. |
| `role` | string | yes | Human-readable role in the reconstruction. |
| `notes` | string | yes | Caveats, source notes, or restoration policy notes. |

## Storage Classes

- `git`: committed to the repository as normal source or small metadata.
- `external`: required evidence that should be restored from an external repo-relative artifact bundle.
- `git_lfs`: evidence suitable for Git LFS if adopted later.
- `runtime_generated`: produced by local audit runs and not committed.
- `external_optional`: not needed for minimum validation but useful for broader thesis provenance.

## Minimum Validate-Only Contract

The minimum manifest must include:

- selected B0 graph
- Stage11 graph output
- Stage12 graph output
- B0 rebuilt evaluator report required by validate-only
- canonical allocation, even though it is expected to be present from Git

The checker verifies all rows in the manifest, including `storage_class=git` rows, so hash drift in tracked evidence is detected.

## Runtime Manifests

This artifact bundle manifest is distinct from runtime audit manifests generated by:

```bash
bash scripts/reconstruction/run_frozen_artifact_reconstruction_audit.sh --validate-only
```

Runtime manifests live under:

`artifacts/final_graph/selected_final_graph/rebuild/runs/`

They remain untracked and are not part of the artifact bundle contract.
