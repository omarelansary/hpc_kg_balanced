# Reproducibility Risks

## Risk Table

| Risk | Severity | Evidence | Why It Matters For Thesis Defensibility | Recommended Fix Without Refactoring Code |
| --- | --- | --- | --- | --- |
| Live Wikidata/WDQS endpoint drift | High | `src/archive/hop_discovery.py`, hop support scripts, composition verifier, Phase4 construction, and repair flows use SPARQL/WDQS; logs show 429/504 failures | Rerunning later can produce different candidates, support counts, shortcut verification, graph triples, or failures | Freeze raw WDQS responses or final intermediate JSONL inputs; record query date, endpoint, user-agent, retry parameters, and output hashes. |
| Original absolute paths in logs and manifests | High | Logs reference `/data/horse/ws/omel305g-omel305g-new/hpc_kg_balanced`; Stage11/12 manifests reference `/home/kg_benchmark/runs/...` | The copied refactor workspace cannot be rerun verbatim from those paths | Add a documentation manifest that maps original paths to copied-workspace equivalents and marks missing external paths. |
| Hardcoded or stale script paths | High | Inverse SLURM runners reference `src/build_inverse_alias_topk.py` and `src/llm_classification_inv.py`, while scripts are under `src/inverse_verification_legacy/` | A rerun can fail or execute the wrong script | Document current path mismatches; later introduce wrapper scripts or update runners in a separate refactor task. |
| Missing exact command history for several steps | High | Direct logs found for discovery/support/composition/repairs/pruning; no direct run log found for relation-profile LLM generation, domain/range enrichment, or dashboard allocation export | Some critical outputs cannot be defended as reproducible from source alone | Create run cards for each artifact with command, input hashes, output hashes, timestamp, and operator notes. |
| LLM provenance gaps | High | `classify_relations_pipeline.py` exists and relation profiles are consumed, but production run log/model metadata for `relation_profiles_afterLLM_SecondTime.json` was not found | LLM target filtering affects relation universe and composition targets; model or prompt drift changes results | Preserve prompt, model, temperature, schema, API date, raw responses, and input/output hashes for LLM artifacts. |
| Incomplete inverse LLM evidence | High | `logs/llm_classification_inv_27548189.err` contains many OpenAI 429 insufficient-quota failures; complete shard set not proven | Inverse pattern classification may be incomplete or biased by failed shards | Audit shard outputs and reports; compute completion table by shard before using inverse labels as final. |
| Loose environment specification | High | `requirements.txt` uses ranges; `.venv/` exists; no lockfile found | Package upgrades can alter OpenAI clients, pandas behavior, plotting, NetworkX, or SPARQL handling | Export exact environment with `pip freeze` or lockfile in a future documentation-only environment record. |
| Randomness not centrally manifested | Medium | Some scripts expose seeds, e.g. Phase4 seed 42, repair `--random_seed 13`, deterministic composition sampling; no single run manifest covers all seeds | Even seeded components are hard to audit when seed provenance is split across logs and scripts | Add per-run manifest files that capture seeds and deterministic flags. |
| Generated files under `src/` | High | `src/Pruning graph/stage11_eta_aware_connectivity_repair_full/` contains large run outputs, reports, CSVs, and generated Stage13 artifacts | Source and generated outputs are mixed, making provenance and code review fragile | Leave files in place for now; document proposed future move to `runs/` or `artifacts/final_outputs/`. |
| Space in path | Medium | `src/Pruning graph/` | Commands can break without quoting; scripts may behave differently across shells | Document exact quoted paths; future refactor can move to `src/pruning_graph/` only after provenance freeze. |
| Multiple similar or versioned scripts | High | `hop_support.py`, `hop_support_copy.py`, `hop_support_v2.py`, `hop_support_v3.py`; multiple pruning scripts; duplicate plotting scripts | It is easy to cite or rerun a non-canonical script | Build an evidence-backed entrypoint registry before deleting or consolidating anything. |
| SLURM logs separated from scripts and outputs | Medium | Logs live under `logs/`; outputs live under `data/`, `src/Pruning graph/`, and external paths in manifests | A reader must manually join command, inputs, outputs, and results | Future run folder should place command, config, logs, manifest, reports, and outputs together. |
| Output artifacts without input hashes | High | Most JSONL/CSV outputs lack recorded hashes of consumed inputs | It is difficult to prove which input version produced a result | Add SHA256 manifests for canonical artifacts before refactoring. |
| Candidate pools without retrieval timestamp | High | Candidate and support files are generated from live WDQS; not all artifacts carry retrieval timestamps | Candidate availability and counts can drift over time | Add retrieval timestamps and endpoint metadata beside every candidate pool. |
| Audit outputs without config linkage | Medium | Stage12 and Stage13 reports exist, but some allocation/config paths point to external or stale locations | Audits can be misinterpreted if paired with the wrong graph or allocation | Create a cross-reference table linking graph, allocation, config, and audit report. |
| Graph outputs without allocation linkage | High | Multiple allocation JSONs exist: allsupp50, allsupp8, hop-v3 patched, `bidirectional_allocation_results5k.json` | Final graph defensibility depends on the exact allocation target | Canonicalize one allocation per reported graph in documentation before any code refactor. |
| Online construction output missing from copied workspace | Medium | Trial2 log names `trial2_connected_allocation_sample.watch.triples.jsonl`, but copied workspace contains checkpoint postprocess output instead | The copied workspace may be incomplete relative to original execution | Record missing log-referenced outputs in open questions and avoid treating them as present. |
| Phase II offline execution not confirmed | High | `relation_balanced_kg_pipeline.py` exists, but no matching run log found; config uses `candidate_source_mode: wdqs` | The intended offline quota-aware candidate-pool story is not yet proven by actual run evidence | Present offline pipeline as scaffold/planned unless later run evidence is found. |
| Stage numbering mismatch | Medium | Intended Phase II has eight stages; `relation_balanced_kg_pipeline.py` exposes stage01 through stage07/final-audit | Misnumbering can confuse thesis narrative and reproducibility claims | Document the mismatch explicitly and choose a single numbering scheme later. |
| `.env` present in workspace | Medium | Hidden-file inventory found `.env`; contents intentionally not inspected | Secrets and local credentials can become hidden dependencies | Do not commit `.env`; document required environment variables separately without values. |
| `.venv` present in workspace | Low | Hidden-file inventory found `.venv/` | Inflates inventory and may hide dependency drift | Exclude from forensic artifact counts or document separately. |
| Dirty tracked files and deleted tracked files | High | `git status --short` shows modified and deleted tracked files before documentation | The workspace state is not a clean baseline | Do not clean automatically; record dirty state and take a git patch/status snapshot before future changes. |

## Highest Priority Non-Refactor Fixes

1. Create a canonical artifact manifest that lists the selected final graph, selected allocation JSON, support matrix, composition file, and all input hashes.
2. Create a run provenance table for every high-confidence executed step, including command, timestamp, host/path, input paths, output paths, seed, endpoint, and log path.
3. Audit LLM outputs before relying on them: relation-profile generation needs model/prompt/run metadata, and inverse classification needs shard completion accounting.
4. Freeze live-WDQS-derived intermediate files as thesis inputs so later refactors do not depend on endpoint state.
5. Document path translation from original source paths and `/home/kg_benchmark/runs/...` to this copied workspace.
