# C5-H2 Auxiliary Acceptance And Ranking Design

## Purpose

This document defines whether C5-H2 may proceed from probe evidence to a constrained generator design.

C5-H2 is an experimental auxiliary-connectivity branch. It is not a canonical allocation-faithful graph design by itself. The branch tests whether observed, unallocated local evidence can be added as auxiliary connectivity support so that surplus canonical B0 bridge edges can be pruned while preserving weak connectivity and canonical relation coverage.

No graph is generated by this design.

## Evidence Basis

The C5 H1/H2 probe found:

- H1 allocated observed replacements were feasible for connectivity but did not improve balance.
- H2 observed unallocated auxiliary support produced `546` feasible candidate-cut pairs.
- The greedy H2 non-reuse upper bound selected `151` moves.
- The greedy H2 upper bound had surplus delta `-151` and deficit delta `0`.

The C5.1 candidate-score provenance audit found:

- H2 candidates have no old Phase II numeric score fields.
- H2 candidates do have provenance fields such as `duplicate_provenance_count`, `source_stage`, `provenance_type`, and `source_sha256`.
- H2 candidates come from previously unselected frozen candidate space.
- Old Phase II scores should not be used as the primary H2 ranking criterion.

## Methodological Acceptance

Observed unallocated auxiliary edges are allowed in C5-H2 only under the label:

`auxiliary_unallocated_observed`

These edges are not canonical benchmark triples. They are auxiliary connectivity-support edges. Their purpose is to create a redundant connection across a B0 bridge cut so that a surplus canonical allocated bridge edge can be pruned.

Accounting rules:

- They are included in full graph connectivity.
- They are included in full graph triple count.
- They are excluded from canonical allocation surplus and deficit calculations.
- They are reported separately from canonical allocated triples.
- They must be observed in frozen local evidence.
- They must not be synthetic, live-only, LLM-created, or unverified.

This means a generated C5-H2 graph, if later produced, would be an experimental auxiliary graph, not a pure canonical allocation realization.

## Edge Classes

| Edge class | C5-H2 status | Meaning |
| --- | --- | --- |
| `canonical_allocated_observed` | allowed | Observed triple whose relation is in the canonical 139 allocated relations. |
| `auxiliary_unallocated_observed` | allowed | Observed triple whose relation is outside the canonical allocation and used only as separately accounted connectivity support. |
| `synthetic_pattern_derived` | disallowed | Triple inferred from structural pattern logic rather than preserved observation. Deferred to H3. |
| `live_verified_observed` | disallowed | Triple found by live WDQS or other live source. Deferred unless frozen and audited. |
| `synthetic_unverified` | disallowed | Triple without preserved observation or validated derivation. Not acceptable. |

The first C5-H2 generator policy must allow only `canonical_allocated_observed` and `auxiliary_unallocated_observed`.

## Ranking Function

Because H2 candidates do not carry old Phase II numeric scores, C5-H2 needs a new deterministic ranking function.

Ranking priority:

1. Pass all hard constraints.
2. Reduce target relation surplus for the canonical edge removed.
3. Cross the target bridge cut.
4. Use stronger preserved provenance, especially higher `duplicate_provenance_count`.
5. Prefer stronger source evidence by `source_stage` and `provenance_type`.
6. Prefer smaller auxiliary footprint.
7. Avoid reusing the same target edge.
8. Avoid reusing the same auxiliary candidate edge.
9. Use stable tie-breakers by `cut_id`, target triple lexical order, and candidate triple lexical order.

The ranking function must not rank primarily by old Stage2 or Stage4 scores.

Initial policy values:

| Threshold | Value |

| Probe upper-bound cap for auxiliary edges | 151 |
| Default generator auxiliary cap | 50 |
| `require_surplus_delta_lt` | 0 |
| `require_total_deficit_delta_le` | 0 |
| `require_canonical_relation_coverage` | 139 |
| `require_zero_allocated_relations` | 0 |
| `require_duplicate_triples` | 0 |
| `require_weak_components_with_auxiliary` | 1 |
| `require_no_registry_update_before_decision` | true |

The generator must also report weak component count after removing auxiliary edges. That value is diagnostic and must not be hidden by the auxiliary-connected view.

## Evaluation Policy

A generated C5-H2 graph, if later produced, must report:

- canonical allocated triple count;
- auxiliary unallocated observed edge count;
- full graph triple count;
- canonical relation coverage;
- surplus and deficit over canonical allocated relations only;
- weak connectivity with auxiliary edges included;
- weak connectivity after removing auxiliary edges;
- unallocated relation count;
- auxiliary relation distribution;
- list and SHA256 hash of auxiliary edges;
- list and SHA256 hash of removed canonical edges;
- graph output manifest and standard evaluator report.

The report must preserve separate accounting for canonical and auxiliary edges. A single combined metric is not sufficient.

## Generator Gate

A C5-H2 generator is allowed only after this policy is accepted. The generator must remain constrained:

- no WDQS;
- no LLM;
- no synthetic triples;
- no registry update before graph/report/decision review;
- no claim that auxiliary edges satisfy canonical allocation quotas.

H3 synthetic pattern-derived triples remain deferred.

## Decision Status

C5-H2 is acceptable only as an experimental auxiliary-connectivity branch. It may test whether observed unallocated support can improve the connectedness-vs-balance frontier by enabling surplus canonical bridge-edge pruning.

It is not acceptable to present C5-H2 as a canonical allocation-faithful graph unless auxiliary edges are excluded from canonical allocation accounting and clearly labelled in all reports.

## Safe Claims

- C5-H2 has probe evidence for observed unallocated auxiliary support.
- C5-H2 auxiliary edges can be evaluated as connectivity support distinct from canonical allocated triples.
- Old Phase II scores are unavailable for H2 rows and should not be the primary ranking criterion.
- A constrained generator can be designed around deterministic hard constraints, surplus reduction, cut crossing, and provenance strength.

## Unsafe Claims

- Do not claim C5-H2 already produced a graph candidate.
- Do not claim auxiliary unallocated edges are canonical benchmark triples.
- Do not count auxiliary unallocated edges toward canonical allocation surplus/deficit.
- Do not rank primarily by old Stage2/Stage4 scores.
- Do not enable synthetic, live WDQS, or LLM-created triples in the first C5-H2 generator.
